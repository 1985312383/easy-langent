# 🎭 剧本杀智能体游戏

基于 LangGraph 构建的 AI 驱动剧本杀游戏，支持用户与 3 个 AI 玩家共同参与一场完整的推理剧本杀对局。

## 🎯 核心特性

- **完整剧本生成**：AI 自动生成案件背景、嫌疑人设定、线索与真相，每局剧本独一无二
- **多角色智能体**：包含 DM、剧本生成、身份分配、线索分配、AI 玩家等多个智能体协同工作
- **用户深度参与**：你将作为玩家之一，参与自我介绍、线索讨论与最终投票
- **AI 真实交互**：3 个 AI 玩家会根据自己的剧本隐藏秘密、分析线索并进行推理发言
- **清晰游戏流程**：严格遵循剧本杀经典流程，环节分明，节奏可控
- **完整对局记录**：所有发言、线索、投票与真相都会完整呈现，复盘清晰

------

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置 LLM

创建 `.env` 文件，配置你的大模型接口（兼容 OpenAI 格式）：

```
API_KEY="your_api_key_here"
```

项目默认使用 DeepSeek 接口，可根据需要修改代码中的 `base_url` 和 `model` 参数。

### 3. 启动游戏

唯一入口：

在bash中运行

```bash
python Scripted_murder_game.py
```

## 🎮 游戏流程详解

### 阶段 1：剧本生成

剧本生成智能体自动生成完整案件剧本，包含：

- 案件背景（现代都市豪宅谋杀案）
- 4 位嫌疑人的身份、背景、秘密与行动轨迹
- 4 条关键线索
- 完整案件真相（凶手身份、动机与作案手法）

### 阶段 2：DM 案件介绍

DM 智能体向所有玩家介绍案件背景、嫌疑人名单与游戏规则，引导玩家进入游戏。

### 阶段 3：身份分配

随机分配 4 个嫌疑人身份，其中 1 个为用户玩家，其余 3 个为 AI 智能体玩家。

### 阶段 4：剧本分发

DM 向每位玩家分发专属剧本，包含身份信息、案发行动轨迹与需要隐藏的秘密。

### 阶段 5：自我介绍环节

玩家依次介绍自己的身份与案发当天行动，AI 玩家会根据剧本隐藏关键秘密，保持言行一致。

### 阶段 6：线索公布与分配

DM 公布 4 条关键线索

线索分配智能体将线索随机分配给 4 位玩家

### 阶段 7：线索讨论环节

每位玩家描述自己获得的线索，并分享对线索的看法，AI 玩家会结合其他玩家的发言进行推理分析。

### 阶段 8：投票环节

所有玩家根据发言与线索，投票选出自己怀疑的凶手，并阐述推理理由。

### 阶段 9：真相揭晓

DM 统计投票结果，公布得票最高的嫌疑人

公布完整案件真相，揭露真凶身份、作案动机与手法

揭晓所有玩家的隐藏秘密，复盘整个案件

## 📊 设计思路

所以我为了简单呈现效果，就先采用一种类型的剧本杀：推理剧本杀。

剧本杀流程：

项目名称：剧本杀智能体项目

技术：langgraph

DM智能体x1、用户玩家x1、智能体玩家x3、剧本生成智能体、身份分配智能体、线索分配智能体

大致思路：先是剧本杀剧本生成智能体生成每个人的相关剧本以及完整的剧本真相，然后DM智能体先自我介绍欢迎大家来到推理剧本杀游戏，接下来DM智能体讲述相关的案情（一名剧本人物遇害，每个玩家都与他有关联，现在找出凶手）然后玩家和4个智能体玩家开始游戏，先是阅读各自剧本，然后环节1：每个玩家分别自我介绍，讲述自己在案发时间都干了些什么（每个人剧本中都包含一段需要隐藏的秘密，在介绍环节可以隐藏），环节2：DM智能体提供相关线索（但这个线索跟秘密相关），但是搜证这个很难实现，毕竟这是对话版，所以换成每个人都可以随机抽取不同 的线索，而这些线索与每个玩家的秘密有关联，通过上一轮的谈话，加上所获取的线索推断那个玩家的说法存在漏洞，进而怀疑谁最可疑，环节3：最后每个人都讲述谁是凶手，并给出理由或自己推理出的真相，环节4：最后投票选出谁是凶手，环节5：DM智能体讲述最终的真相，游戏结束。提示词要求：符合需求，逻辑严谨闭环。

具体节点：

节点1：剧本生成智能体生成剧本信息

节点2：DM智能体接收剧本生成智能体的信息，介绍案件剧本信息

节点3：身份分配智能体分配用户玩家和3个AI智能体对应剧本身份

节点4：DM智能体分配给4个玩家剧本信息（人物关系、需要隐藏的秘密）

节点5：玩家用户输入自己的发言

节点6：玩家智能体1根据自己的剧本要求发言

节点7：玩家智能体2根据自己的剧本要求发言

节点8：玩家智能体3根据自己的剧本要求发言

节点9：DM智能体给出相关的4条线索

节点10：线索分配智能体随机分配线索给4个玩家

节点11：玩家用户描述自己的线索，玩家智能体依次描述自己的线索

节点12：投票环节，玩家用户选择输入怀疑的玩家为凶手并阐述理由，玩家智能体也依次选择输入怀疑的玩家并阐述理由

节点13：最后DM智能体统计投票结果，并公布正确答案凶手，并阐述真相，游戏结束。
